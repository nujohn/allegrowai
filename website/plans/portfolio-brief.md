- Public URL: `https://baystrings.org`

## Design direction
- Fun but still credible and polished
- Not sterile SaaS, not childish
- Should feel like a founder / builder portfolio, not a generic agency template
- Should make the work look thoughtful, crafted, and real

## Content direction
Likely sections:
- Hero / positioning
- Featured projects
- Product snapshots / screenshots
- Build philosophy / approach
- Optional about / contact / future work section

## Maintainability requirement
Site should be easy to extend as more projects are added.
Prefer:
- clear content/data-driven project structure
- reusable project card / feature section components
- easy screenshot / logo / metadata updates
- plans/docs saved in repo
